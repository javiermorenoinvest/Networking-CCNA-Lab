# Wireshark Intro Lab

## What I Worked On

Today I started learning the basics of Wireshark.

Wireshark is a network packet analyzer. It lets you capture and inspect network traffic so you can see what devices are sending and receiving across a network.

## What I Did

- Looked up what Wireshark is used for
- Downloaded and installed Wireshark
- Opened Wireshark for the first time
- Captured a small amount of network traffic
- Clicked through a few packets to see what information they contained
- Started getting familiar with how packet captures are displayed

## What I Observed

When I captured traffic, I saw that Wireshark shows packets in a detailed list. Each packet included information such as:

- Source address
- Destination address
- Protocol
- Packet length
- Basic packet details

I did not fully understand everything yet, but I started recognizing that Wireshark lets you look “under the hood” of network communication.

## Why This Matters

Learning Wireshark is important because it helps build real networking and cybersecurity skills. Instead of only reading about network traffic, I can actually see packets moving across a network.

This is useful for:

- Troubleshooting network issues
- Understanding protocols
- Learning how devices communicate
- Building packet analysis skills
- Preparing for IT and cybersecurity roles

## What I Learned

I learned that Wireshark is not the same thing as Cisco Packet Tracer.

Packet Tracer is used to simulate networks.

Wireshark is used to capture and analyze real network traffic.

That helped me understand why browsing the web does not automatically show up inside Packet Tracer. Packet Tracer is a lab simulation, while Wireshark watches real traffic from my actual computer or network interface.
